# Test Plan - OrangeHRM Manual Testing Project

## 1. Project Overview
This Test Plan describes the testing approach, scope, objectives, and activities performed for testing the OrangeHRM Demo application.

## 2. Application Under Test (AUT)
OrangeHRM Demo Application

## 3. Testing Scope

### In Scope:
- Login functionality
- Dashboard verification
- Admin module
- PIM module
- Employee Management
- Leave Management

### Out of Scope:
- Performance Testing
- Security Testing
- Database Testing

## 4. Testing Objectives
- Verify application functionality.
- Identify defects and report them.
- Ensure the application meets expected requirements.
- Validate user workflows.

# 5. Testing Strategy
The following testing approaches will be performed:
- Functional Testing
- Smoke Testing
- Regression Testing
- UI Testing

# 6. Test Environment
- Browser: Google Chrome
- Operating System: Windows
- Testing Type: Manual Testing

# 7. Test Deliverables
- Test Scenarios
- Test Cases
- Bug Reports
- Test Summary Report

# 8. Entry Criteria
- Application is available for testing.
- Requirements are understood.
- Test environment is ready.

# 9. Exit Criteria
- All planned test cases are executed.
- Critical defects are reported.
- Test summary report is prepared.

# 10. Risks
- Environment availability issues.
- Changes in application requirements.
- Limited testing time.
